NexusFlow Skill ​

NexusFlow Skill is the separately published instruction package that helps Cursor, Claude Code, and other skills.sh / AGENTS-compatible tools use the MCP server exposed by the NexusFlow desktop app more reliably.

Repository: https://github.com/maxzhao/nexusflow-skill

Install:

bash
npx skills add maxzhao/nexusflow-skill

It is not part of the NexusFlow desktop app, and it is not a plugin for connecting external Skills into NexusFlow. Its purpose is to teach external IDEs and agents how to use NexusFlow MCP tools safely.

Requirements ​

Before installing or using NexusFlow Skill, make sure the NexusFlow runtime is ready:

  1. The NexusFlow desktop app is installed and running.
  2. MCP mode is enabled in NexusFlow.
  3. Your IDE or MCP client is connected to the NexusFlow MCP endpoint.
  4. For UE operations, Unreal Engine is open and the NexusFlow UE plugin is installed and connected.

If this setup is not ready yet, start with Use NexusFlow from IDEs / Agents.

Why It Exists ​

When an external agent sees MCP tools directly, it may not know that it should:

  • Check whether NexusFlow is connected to UE first.
  • Inspect the current UE context instead of guessing asset or Blueprint state.
  • Search and load a NexusFlow Skill guide before executing actions.
  • Load knowledge topics only when needed.
  • Treat direct Python execution as a confirmed last resort.

NexusFlow Skill packages these rules as installable agent instructions so agents can operate UE with a safer and more stable workflow.

Installation ​

In a skills.sh-compatible environment, run:

bash
npx skills add maxzhao/nexusflow-skill

If GitHub access is unstable in your network, configure proxy settings for Git / npm. Example:

bash
ALL_PROXY=socks5://127.0.0.1:10808 \
HTTPS_PROXY=http://127.0.0.1:10808 \
HTTP_PROXY=http://127.0.0.1:10808 \
npx skills add maxzhao/nexusflow-skill

Entrypoints ​

After installation, the Skill repository contains entrypoints for different agents:

FileEnvironment
SKILL.mdUniversal Skill entrypoint
AGENTS.mdAgents or IDEs that read AGENTS instructions
CLAUDE.mdClaude Code
.cursor/rules/nexusflow.mdcCursor
references/mcp-tools.mdDetailed MCP tool reference

In most cases, you only need to install it through skills.sh; the target tool reads the matching file.

Agent Workflow ​

After installation, agents should prefer this progressive workflow:

text
get_connection_status
  -> get_ue_context
  -> search_skills
  -> load_skill
  -> [load_knowledge]
  -> execute_action

Core rules:

  • The live tool list and schemas shown by the MCP client are the source of truth.
  • Call get_connection_status before any UE or Blueprint operation.
  • After a successful connection check, call get_ue_context instead of guessing editor state.
  • Prefer searching and loading NexusFlow Skills over direct Python execution.
  • Do not repeatedly load the same Skill, knowledge topic, or template in one conversation.
  • Verify write operations after execution.
  • Stop immediately on USER_CANCELLED.

Who Should Use This ​

  • Developers who want AI in Cursor to operate UE Blueprints.
  • Claude Code users who want to use the NexusFlow MCP toolchain.
  • Teams integrating NexusFlow as an Unreal Engine automation gateway for custom agents.
