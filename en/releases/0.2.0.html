NexusFlow 0.2.0 Beta ​

0.2.0 is the first Beta update after 0.1.0. It expands NexusFlow from a desktop AI Blueprint assistant into an Unreal Engine automation gateway for external IDEs and agents, while improving Blueprint reliability and introducing an experimental Widget Blueprint capability.

Highlights ​

Embedded MCP Server ​

The NexusFlow desktop app can now run as an MCP server. MCP-capable IDEs and agents can connect to NexusFlow and use UE context, Skill search, Knowledge loading, Blueprint action execution, node search, template search, and editor commands.

Agent Skill Instructions ​

The new nexusflow-skill package provides portable entrypoints for AGENTS, Claude Code, and Cursor, teaching external agents how to use NexusFlow MCP tools safely.

Blueprint Skill Reliability Improvements ​

Blueprint read, generation, incremental patching, compile validation, and error recovery workflows have been improved. Handling is more reliable for complex graphs, variables, components, interfaces, dynamic pins, and special nodes.

Widget Skill MVP ​

The experimental Widget Skill can create and edit UMG Widget Blueprints, including widget tree read/write, property-level patching, branch rewrites, and real preview screenshots.

Agent-Friendly Search Workflow ​

Blueprint template search, template loading, and node search are exposed as MCP tools so agents can discover reusable patterns and correct nodes before building Blueprint logic.

Environment Variable API Key References ​

Model configuration now accepts environment variable references such as ${OPENAI_API_KEY} in the API Key field. The real key is resolved by the Tauri backend at runtime, while the app configuration only stores the variable reference.

Upgrade Notes ​

  • If you only use the NexusFlow desktop chat UI, you can upgrade normally.
  • If you prefer not to store the real API key in app configuration, enter ${ENV_NAME} in the API Key field, for example ${OPENAI_API_KEY}.
  • If you use Cursor, Claude Code, or custom agents, read Use NexusFlow from IDEs / Agents and NexusFlow Skill.
  • Widget Skill is still experimental; start with small UIs or assets that are easy to roll back.
