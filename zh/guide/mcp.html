在 IDE / Agent 中使用 NexusFlow ​

NexusFlow 0.2.0 在桌面主程序中内置 MCP Server。启用后，Cursor、Claude Code 或其他支持 MCP 的 IDE / Agent 可以连接到 NexusFlow，把 NexusFlow 当作 Unreal Engine 自动化能力入口来使用。

这不是“把外部 MCP 接入 NexusFlow 主程序”的功能；方向相反：外部 IDE / Agent 连接 NexusFlow 提供的 MCP Server，然后通过 NexusFlow 操作 UE 上下文、Blueprint、Widget、模板和节点搜索等能力。

适用场景 ​

  • 在 Cursor、Claude Code 或其他 MCP Client 中让 Agent 直接操作 UE 蓝图。
  • 让外部 IDE 读取当前 UE 编辑器上下文，例如选中资产、打开的蓝图和连接状态。
  • 使用 NexusFlow Skills 执行受约束的蓝图、Widget、资产等操作。
  • 在写蓝图逻辑前搜索 UE 节点或复用 Blueprint 模板。

前置条件 ​

使用前，请确认：

  1. 已安装并启动 NexusFlow 桌面应用。
  2. 已在 NexusFlow 中启用 MCP 模式。
  3. MCP Client 已连接到 NexusFlow 提供的 MCP endpoint。
  4. 如需执行 UE 操作，Unreal Engine 已打开，NexusFlow UE 插件已安装并连接。

如果 MCP Client 无法连接，请先打开 NexusFlow，确认 MCP 模式已启用，并检查 Client 中配置的 endpoint。

推荐配套安装 NexusFlow Skill ​

为了让 IDE / Agent 更稳定地使用 NexusFlow MCP，建议安装独立发布的 NexusFlow Skill：

bash
npx skills add maxzhao/nexusflow-skill

它会告诉 Agent 如何检查连接、读取 UE 上下文、搜索并加载 NexusFlow Skills，以及如何安全执行动作。

推荐工作流 ​

外部 Agent 应使用渐进式披露流程，避免一次性加载过多上下文：

text
get_connection_status
  -> get_ue_context
  -> search_skills
  -> load_skill
  -> [load_knowledge]
  -> execute_action

说明：

  • get_connection_status：任何 UE 操作前先检查连接，只在 connected: true 时继续。
  • get_ue_context：读取当前 UE 编辑器状态，理解选中资产、打开蓝图和当前上下文。
  • search_skills：根据用户目标搜索最相关的 NexusFlow Skill。
  • load_skill：加载 Skill 指南，查看可用函数、参数和约束。
  • load_knowledge：仅在指南要求或任务需要时加载具体知识主题。
  • execute_action：按已加载指南构造参数并执行动作。

可用 MCP 工具 ​

工具名称可能因 MCP Client 配置而带有不同前缀，请以 Client 显示的实时 tool list 和 schema 为准。

工具用途
get_connection_status检查 NexusFlow 是否已连接 UE
get_ue_context获取当前 Unreal Editor 上下文
search_skills搜索 NexusFlow Skill 模块
load_skill加载 Skill 指南和可用知识主题
load_knowledge加载指定 Skill 的专题知识
execute_action执行 Skill 函数
search_templates搜索可复用 Blueprint 模板
load_template加载模板完整定义
search_nodes搜索 Blueprint 节点和引擎函数
execute_command执行受支持的编辑器或控制台命令
execute_python_code最后手段：执行直接 Python，需要明确确认

安全建议 ​

  • 优先使用 search_skills / load_skill / execute_action，不要直接猜 UE Python API。
  • 不要重复加载同一个 Skill、知识主题或模板。
  • 对已有资产做局部、增量修改；完整重写或破坏性操作前必须确认。
  • 写操作后通过返回结果、读回、编译、保存或截图验证。
  • execute_python_code 只作为确认后的最后手段。
