NexusFlow Skill ​

NexusFlow Skill 是独立发布的 Agent 使用说明包，用于帮助 Cursor、Claude Code 以及其他支持 skills.sh / AGENTS 说明的工具，更稳定地使用 NexusFlow 桌面应用提供的 MCP Server。

仓库地址：https://github.com/maxzhao/nexusflow-skill

安装命令：

bash
npx skills add maxzhao/nexusflow-skill

它不是 NexusFlow 桌面主程序的一部分，也不是把外部 Skill 接入 NexusFlow 主程序的插件。它的作用是：让外部 IDE / Agent 知道如何安全使用 NexusFlow MCP 工具。

使用前提 ​

安装 NexusFlow Skill 前，请先准备好 NexusFlow 运行环境：

  1. 已安装并启动 NexusFlow 桌面应用。
  2. 已在 NexusFlow 中启用 MCP 模式。
  3. 你的 IDE / MCP Client 已连接到 NexusFlow MCP endpoint。
  4. 如需执行 UE 操作，Unreal Engine 已打开，NexusFlow UE 插件已安装并连接。

如果这些条件还没满足，请先阅读 在 IDE / Agent 中使用 NexusFlow。

它解决什么问题 ​

外部 Agent 直接看到 MCP 工具时，可能不知道：

  • 应该先检查 NexusFlow 与 UE 是否连接；
  • 应该先读取当前 UE 上下文，而不是猜测资产或蓝图状态；
  • 应该先搜索并加载 NexusFlow Skill 指南，再执行动作；
  • 什么时候需要加载知识主题；
  • 为什么直接执行 Python 应该作为最后手段。

NexusFlow Skill 把这些规则写成可安装的 Agent 指令，帮助 Agent 按稳定、安全的流程完成 UE 操作。

安装 ​

在支持 skills.sh 的环境中运行：

bash
npx skills add maxzhao/nexusflow-skill

如果你所在网络访问 GitHub 不稳定，需要为 Git / npm 配置代理。例如：

bash
ALL_PROXY=socks5://127.0.0.1:10808 \
HTTPS_PROXY=http://127.0.0.1:10808 \
HTTP_PROXY=http://127.0.0.1:10808 \
npx skills add maxzhao/nexusflow-skill

包含的入口 ​

安装后，Skill 仓库中包含面向不同 Agent 的入口文件：

文件适用环境
SKILL.md通用 Skill 入口
AGENTS.md读取 AGENTS 指令的 Agent / IDE
CLAUDE.mdClaude Code
.cursor/rules/nexusflow.mdcCursor
references/mcp-tools.mdMCP 工具细节参考

多数情况下，你只需要通过 skills.sh 安装；具体文件由对应工具读取。

Agent 应遵循的流程 ​

安装后，Agent 应优先使用以下渐进式流程：

text
get_connection_status
  -> get_ue_context
  -> search_skills
  -> load_skill
  -> [load_knowledge]
  -> execute_action

关键规则：

  • MCP Client 显示的实时 tool list 和 schema 是最终事实来源。
  • 任何 UE 或 Blueprint 操作前都要先调用 get_connection_status。
  • 连接成功后调用 get_ue_context，不要脱离 UE 当前上下文猜测。
  • 优先搜索并加载 NexusFlow Skill，而不是直接执行 Python。
  • 同一会话不要重复加载同一个 Skill、知识主题或模板。
  • 写操作后必须验证结果。
  • 遇到 USER_CANCELLED 立即停止。

适合谁使用 ​

  • 希望在 Cursor 中让 AI 操作 UE 蓝图的开发者。
  • 希望在 Claude Code 中使用 NexusFlow MCP 工具链的技术用户。
  • 想把 NexusFlow 当作 UE 自动化 MCP 网关接入自定义 Agent 的团队。
