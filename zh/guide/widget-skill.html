Widget Skill ​

Widget Skill 是 NexusFlow 0.2.0 中可试用的 UMG Widget Blueprint 能力。它面向 HUD、面板、菜单、状态栏等 UI 资产，帮助 Agent 通过结构化定义创建和修改 Widget Blueprint。

当前为 MVP / 实验性能力。建议从小型 UI、局部修改和可读回验证的任务开始使用。

能做什么 ​

  • 从零创建 Widget Blueprint。
  • 读取已有 Widget Blueprint 的结构和布局信息。
  • 修改已有控件的文本、颜色、百分比、slot、事件声明或绑定声明。
  • 重写完整 Designer 或某个容器分支。
  • 在可用时捕获真实 Widget 预览截图。
  • 与 Blueprint Skill 配合，为 Widget 添加事件、变量、函数和 EventGraph 逻辑。

主要动作 ​

动作用途
generate_widget_blueprint创建完整 Widget Blueprint
read_widget_blueprint读取结构、布局和诊断信息
write_widget_tree替换完整 Designer 或指定容器分支
patch_widget_tree修改已有控件的属性级信息
capture_widget_preview捕获真实预览截图

推荐流程 ​

text
1. 确认目标 UI 类型和 viewport 尺寸。
2. 用根 Canvas 放少量区域容器，例如 Header、Content、Footer、Modal。
3. 区域内部使用 Border、VerticalBox、HorizontalBox、ScrollBox、SizeBox 等容器排版。
4. 为重要控件设置稳定 id；交互控件设置 variable.is_variable=true。
5. 执行 generate_widget_blueprint 或 write_widget_tree。
6. 使用 read_widget_blueprint 读回结构与布局诊断。
7. 如需事件或绑定，再用 Blueprint Skill 补充 Graph 逻辑。
8. 编译、保存或截图验证。

使用建议 ​

  • Widget Tree 和 Blueprint Graph 是两层概念：Designer 控件写在 widget_tree，事件、函数、变量和图逻辑写在 Blueprint 通用定义中。
  • Canvas 只适合放区域容器，不建议把大量 TextBlock/Button 直接散放在根 Canvas 下。
  • patch_widget_tree 适合属性级小改；层级变化请使用 write_widget_tree。
  • 写入成功不等于最终状态正确，必须通过 read_widget_blueprint 或截图读回验证。
  • 复杂 UI 建议先生成静态结构，再逐步添加交互逻辑。

  • 蓝图操作：了解通用 Blueprint Graph 能力。
