NexusFlow 0.2.0 Beta ​

0.2.0 是 0.1.0 之后的第一个 Beta 更新，重点是把 NexusFlow 从“桌面 AI 蓝图助手”扩展为“可被外部 IDE / Agent 接入的 UE 自动化网关”，同时继续提升蓝图操作稳定性，并引入可试用的 Widget Blueprint 能力。

亮点 ​

内置 MCP Server ​

NexusFlow 桌面主程序现在可以作为 MCP Server 运行。支持 MCP 的 IDE 或 Agent 可以连接 NexusFlow，使用 UE 上下文、Skill 搜索、Knowledge 加载、Blueprint 动作执行、节点搜索、模板搜索和编辑器命令等能力。

NexusFlow Skill ​

新增 nexusflow-skill，提供 AGENTS、Claude Code 和 Cursor 的便携入口，让外部 Agent 知道如何安全使用 NexusFlow MCP 工具。

Blueprint Skill 稳定性提升 ​

蓝图读取、生成、增量 patch、编译验证和错误恢复流程得到大量修补。对复杂图、变量、组件、接口、动态 pin 和特殊节点的处理更可靠。

Widget Skill MVP ​

新增实验性的 Widget Skill，可创建和修改 UMG Widget Blueprint，支持 Widget Tree 读写、属性级 patch、分支重写和真实预览截图。

更适合 Agent 的搜索工作流 ​

Blueprint 模板搜索、模板加载和节点搜索已作为 MCP 工具暴露，便于 Agent 在构造蓝图逻辑前先发现可复用模式和正确节点。

API Key 环境变量引用 ​

模型配置现在支持在 API Key 字段填写 ${OPENAI_API_KEY} 这类环境变量引用。真实密钥由 Tauri 后端在运行时读取，配置中只保存变量名引用，适合不想把明文 API Key 写入应用配置的用户。

全局 LLM 代理 ​

设置中心新增独立的「LLM 代理」页面，可为所有 LLM 请求统一配置 HTTP / HTTPS / SOCKS4 / SOCKS5 代理。代理密码使用加密密钥存储保存；该代理只作用于 LLM 请求，不影响 UE、Bridge、MCP 或窗口内部通信。

升级建议 ​

  • 如果你只使用 NexusFlow 桌面聊天窗口，可以照常升级。
  • 如果你希望避免在应用配置中保存真实 API Key，可以在模型配置的 API Key 字段填写 ${ENV_NAME}，例如 ${OPENAI_API_KEY}。
  • 如果你的网络环境访问 LLM 服务需要代理，可以在设置 → LLM 代理中配置全局代理；若本地代理软件启用了分流/绕过规则，请确保目标 LLM 服务被设置为强制代理。
  • 如果你使用 Cursor、Claude Code 或自定义 Agent，建议阅读 在 IDE / Agent 中使用 NexusFlow 和 NexusFlow Skill。
  • Widget Skill 仍为实验性能力，建议从小型 UI 或可回滚资产开始试用。
