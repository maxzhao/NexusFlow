Unreal Engine Blueprint T3D 格式研究报告 ​

概述 ​

T3D (Text 3D) 是 Unreal Engine 用于序列化对象的文本格式。当你在蓝图编辑器中复制节点时， 剪贴板中的内容就是 T3D 格式。本文档详细分析蓝图节点的 T3D 表示形式。

数据来源:

  • UE 源码: Engine/Source/Editor/UnrealEd/Private/Kismet2/
  • UE 源码: Engine/Source/Editor/BlueprintGraph/
  • 实际复制蓝图节点获取的 T3D 样本

1. T3D 格式完整结构 ​

1.1 基本语法 ​

text
Begin Object Class=/Script/BlueprintGraph.K2Node_CallFunction Name="K2Node_CallFunction_0"
   PropertyName=PropertyValue
   StructProperty=(Field1=Value1,Field2=Value2)
   ArrayProperty(0)=Value0
   ArrayProperty(1)=Value1
End Object

1.2 通用字段 (所有节点类型共享) ​

必需字段 ​

字段类型说明
ClassPath节点类的完整路径，如 /Script/BlueprintGraph.K2Node_CallFunction
NameString节点实例名称，如 K2Node_CallFunction_0
NodeGuidGUID节点唯一标识符，格式: XXXXXXXXXXXXXXXXXXXXXXXXXXXXXXXX
NodePosXInteger节点在图表中的 X 坐标
NodePosYInteger节点在图表中的 Y 坐标

可选字段 ​

字段类型默认值说明
NodeCommentString空节点注释文本
bCommentBubblePinnedBooleanFalse注释气泡是否固定显示
bCommentBubbleVisibleBooleanFalse注释气泡是否可见
AdvancedPinDisplayEnumHidden高级引脚显示状态
EnabledStateEnumEnabled节点启用状态
bUserSetEnabledStateBooleanFalse用户是否手动设置启用状态
ErrorTypeInteger0错误类型
ErrorMsgString空错误消息

1.3 EnabledState 枚举值 ​

cpp
enum ENodeEnabledState
{
    Enabled,              // 正常启用
    Disabled,             // 禁用 (灰色显示)
    DevelopmentOnly       // 仅开发模式 (紫色显示)
};

1.4 AdvancedPinDisplay 枚举值 ​

cpp
enum ENodeAdvancedPins
{
    NoPins,    // 无高级引脚
    Hidden,    // 高级引脚隐藏
    Shown      // 高级引脚显示
};

3. Pin 的 T3D 表示 ​

3.1 Pin 基本结构 ​

Pin 使用 CustomProperties Pin 语法表示：

text
CustomProperties Pin (PinId=GUID,PinName="name",PinFriendlyName="显示名",PinToolTip="提示",Direction="EGPD_Input|EGPD_Output",PinType.PinCategory="category",...)

3.5 Pin 连接信息 ​

连接通过 LinkedTo 字段存储，格式为：

text
LinkedTo=(NodeGuid=TARGETGUID,PinId=TARGETPINID,)

完整示例:

text
CustomProperties Pin (PinId=ABC123,PinName="ReturnValue",Direction="EGPD_Output",PinType.PinCategory="float",LinkedTo=(K2Node_CallFunction_1 GUID1,))

多连接示例 (一个输出连接多个输入):

text
LinkedTo=(K2Node_CallFunction_1 GUID1,K2Node_IfThenElse_0 GUID2,)

4. 特殊属性表示 ​

4.1 NodeGuid ​

32 位十六进制字符串，无分隔符：

text
NodeGuid=A1B2C3D4E5F6789012345678ABCDEF01

源码定义 (EdGraphNode.h):

cpp
UPROPERTY()
FGuid NodeGuid;

4.2 NodeComment ​

节点注释，支持多行 (使用 \n 转义):

text
NodeComment="这是注释\n第二行"
bCommentBubblePinned=True
bCommentBubbleVisible=True

4.3 AdvancedPinDisplay ​

控制高级引脚显示状态：

text
AdvancedPinDisplay=Shown

可选值: NoPins, Hidden, Shown

4.4 EnabledState ​

节点启用状态：

text
EnabledState=DevelopmentOnly
bUserSetEnabledState=True

可选值: Enabled, Disabled, DevelopmentOnly

4.5 自定义属性 (CustomProperties) ​

除 Pin 外，还可存储其他自定义属性：

text
CustomProperties UserDefinedPin (PinName="MyParam",PinType=(PinCategory="float"),DesiredPinDirection=EGPD_Input)

用于 CustomEvent 的用户定义参数:

text
Begin Object Class=/Script/BlueprintGraph.K2Node_CustomEvent Name="K2Node_CustomEvent_0"
   CustomFunctionName="MyEvent"
   CustomProperties UserDefinedPin (PinName="Damage",PinType=(PinCategory="float"),DesiredPinDirection=EGPD_Output)
   CustomProperties UserDefinedPin (PinName="Instigator",PinType=(PinCategory="object",PinSubCategoryObject="/Script/Engine.Actor"),DesiredPinDirection=EGPD_Output)
End Object

7. 解析注意事项 ​

7.1 字符串转义 ​

  • 双引号内的字符串使用 \" 转义
  • 换行使用 \n
  • 反斜杠使用 \\

7.2 GUID 格式 ​

  • 32 位十六进制，无分隔符
  • 示例: A1B2C3D4E5F6789012345678ABCDEF01

7.3 路径格式 ​

  • 完整路径: /Script/Engine.Actor
  • 资产路径: /Game/MyBlueprint.MyBlueprint
  • 子对象路径: /Game/MyBlueprint.MyBlueprint:MacroName

7.4 数组索引 ​

  • 使用括号: ArrayProperty(0)=Value
  • Pin 名称: [0], [1], [2]

7.5 结构体语法 ​

text
StructProperty=(Field1=Value1,Field2=Value2,NestedStruct=(A=1,B=2))

7.6 空值表示 ​

  • 空字符串: ""
  • 空对象: None
  • 空数组: 不出现该属性

9. 总结 ​

关键发现 ​

  1. 统一基础结构: 所有节点共享 Begin Object...End Object 语法和通用字段
  2. 类型差异化: 不同节点类型通过特有字段区分功能
  3. Pin 是核心: 节点的输入输出完全由 CustomProperties Pin 定义
  4. 连接存储: 连接信息存储在 Pin 的 LinkedTo 字段中
  5. 动态 Pin: 通过 NumInputs 等字段控制动态 Pin 数量

解析建议 ​

  1. 先解析 Begin Object 行获取 Class 和 Name
  2. 根据 Class 确定节点类型和特有字段
  3. 解析通用字段 (NodeGuid, NodePosX/Y 等)
  4. 解析 CustomProperties Pin 获取所有 Pin
  5. 从 LinkedTo 重建连接关系
